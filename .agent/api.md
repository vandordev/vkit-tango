# API Rules

`apps/api` is an Uber Fx composition root. Huma with Chi through `humachi` registers business operations only through `/api/v1`; health, readiness, docs, and OpenAPI remain process routes. One handler file owns one method-plus-path operation and validates/maps transport input before calling one typed `internal/contract.Command`. A mutation calls one command. Public payloads use named strict Go structs with explicit fields and JSON tags, not `any`, `interface{}`, or `map[string]any` by default.

A mutation invokes an intent-named Go use case; an Ent read may shape a response directly. API handlers do not own transactions, persistence mutations, River enqueueing, or business policy. Every operation has a deterministic OperationID, concise summary, and one domain tag. Map domain errors to stable public codes without leaking SQL, secrets, or internal topology; review non-additive OpenAPI changes with their Hey API consumers. Generated HTTP registration belongs in `internal/generated/fx` and is refreshed with `task sync:http` or `task sync`; do not hand-maintain provider or route lists. Goose owns schema history, River owns asynchronous work, Hey API consumes the Huma contract, and Socket.IO is only a realtime invalidation boundary, never a write path.
