# Database Rules

Ent schema source is `database/schema`, generated output is `internal/platform/db`, and Goose migrations are the deployed schema history. Model closed values with `field.Enum`; timestamps are UTC `time.Time`, while date-only and time-only values use distinct domain types. JSON uses typed structs or slices; Schema-less JSON is an explicit exception rather than a default. Change Ent schema, regenerate, add a Goose migration, and use an expand/migrate/contract rollout plus retryable backfill for incompatible data changes. Go usecases own Ent writes and River enqueueing in one SQL transaction. Huma exposes writes under `/api/v1`; Hey API transports reads to web; Socket.IO only signals refetches.
